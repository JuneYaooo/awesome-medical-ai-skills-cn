## medical-guidelines-suite ✅ 国内直接可用

- **仓库**: https://github.com/fshaan/medical-guidelines-suite
- **平台**: Claude Code / OpenClaw
- **版本**: 2.0.0
- **安装**: `npx skills add fshaan/medical-guidelines-suite -g -y`

### 功能概述

三阶段临床指南工作流：

1. **构建 (BUILD)** — 将 PDF/DOCX 指南 → 提取文本 + 生成搜索索引
2. **查询 (RAG)** — 搜索索引 → 生成临床推荐意见
3. **批量 (BATCH)** — Excel 患者列表 → RAG 检索 → 生成报告

### 支持的指南体系

- NCCN（美国国家综合癌症网络）
- ESMO（欧洲肿瘤内科学会）
- CSCO（中国临床肿瘤学会）
- 可自行添加其他指南

### 输出格式

| 文件 | 说明 |
|------|------|
| `批量推荐汇总表.xlsx` | 所有患者+指南推荐汇总 |
| `reports/{ID}_{姓名}_推荐意见书.docx` | 单患者推荐意见书 |
| `批量推荐幻灯片.pptx` | 演示用幻灯片 |

### 使用示例

```
# 单患者查询
"HER2阳性晚期胃癌一线治疗，各指南推荐什么？"

# 批量处理
"对 patients.xlsx 中的患者，批量检索指南推荐"
```

### 国内使用注意

- ✅ 输出语言为**简体中文**
- ✅ 支持 CSCO 中国指南
- ⚠️ 需要自行获取指南 PDF（版权原因不内置）
- ⚠️ 依赖 Claude API（需有可用的 API 访问）

## cyber-doctor（赛博华佗） ✅ 国内直接可用

- **仓库**: https://github.com/Warma10032/cyber-doctor
- **团队**: 社区开发
- **特点**: 可完全本地部署

### 功能

- 基于多模态大模型的多功能医疗智能体
- 接入医疗知识图谱和知识库后支持：
  - 疾病初诊
  - 病历分析
  - 专业知识问答
- 一键搭建本地多模态大模型
- 帮助实现医疗资源的跨地域传播

### 国内优势

- ✅ 支持本地部署，无需外网
- ✅ 中文原生支持
- ✅ 可接入国内知识图谱

## Claude-Ally-Health ✅ 国内直接可用

- **仓库**: https://github.com/huifer/Claude-Ally-Health
- **平台**: Claude Code
- **特点**: 中英双语

- 个人健康档案管理（JSON 文件存储）

- 辐射剂量追踪
- 医疗报告管理

### 国内使用注意

- ✅ 完善的中文文档和界面
- ⚠️ 依赖 Claude API
- ⚠️ 图像识别依赖外部 API
